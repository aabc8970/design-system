Status 使用指南

給 PM 的說明：每個元件都有多種狀態（Status），代表用戶互動後的視覺變化。以下說明每種狀態的觸發條件，方便你在 PRD 或驗收時對齊。
狀態名稱 視覺變化 觸發條件 適用元件
default 原始外觀 頁面剛載入，用戶尚未互動 所有元件
pressed 顏色加深或縮小 用戶手指按下（touchDown），放開後恢復 所有按鈕
disabled 灰色，無法點擊 條件未滿足（如表單未填完）才允許操作 按鈕、輸入框
focused 邊框變品牌色（2pt） 用戶點擊輸入框，鍵盤彈出 所有輸入框
error 邊框變紅 + 錯誤提示文字 用戶送出但驗證不通過；格式錯誤 輸入框
selected 品牌色填滿 用戶主動點擊選取 Chip、Mode Selector
unread 左側綠色豎條 對方有新訊息尚未閱讀 User Row
loading 外觀淡化，光暈旋轉動畫 點擊後等待伺服器回應 Find Button
failed 氣泡變淡紅 + 驚嘆號 icon 訊息送出失敗（網路問題） Chat Bubble Sent

Dos & Don'ts · PM 使用元件的注意事項

✓ Do 每個畫面只放一個綠色 CTA 按鈕，視覺上的主角只能有一個。
✗ Don't 同一個畫面放兩個綠色按鈕，用戶會不知道先點哪個。
✓ Do 按鈕文案用動詞開頭：「Find People」「Start Chat」「Update Profile」
✗ Don't 按鈕文案用名詞或模糊詞：「OK」「Submit」「Click Here」
✓ Do 空狀態頁提供明確行動 CTA，引導用戶做第一步。
✗ Don't 空狀態只顯示「No results」或「Nothing here yet」，沒有後續引導。
✓ Do Toast 訊息 10 字以內，說清楚發生了什麼事。
✗ Don't Toast 顯示技術錯誤訊息如「Error 500: Internal Server Error」
✓ Do 使用現有元件的 status 變體來表達不同狀態。
✗ Don't 隨意改元件顏色或自己創新的視覺狀態，會破壞一致性。
